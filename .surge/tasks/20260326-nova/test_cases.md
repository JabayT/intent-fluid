# 「丝生纪」Test Cases

> **最后更新**：2026-03-26（Iter-01 QA 后更新）

---

## L1 核心标准

| ID | 验收项 | 验证方法 |
|----|--------|----------|
| L1-1 | 自主选定科幻子类型，并说明市场对齐理由 | 检查 Master Concept 是否包含子类型选择及理由 |
| L1-2 | 提出至少一个"第一性原理"创新概念（X-Factor） | 检查创新概念是否存在且非主流已有 |
| L1-3 | 章节大纲不少于5章，每章有标题和情节目标 | 计数并验证 |
| L1-4 | 每章标注叙事功能（铺垫/递进/高潮等） | 检查标注完整性 |
| L1-5 | 核心角色不少于3个，含简要描述 | 计数并验证 |
| L1-6 | 全部交付物为中文 | 语言检查 |
| L1-7 | 无明显事实/逻辑错误 | 通读检查 |

## L2 质量标准

| ID | 验收项 | 验证方法 |
|----|--------|----------|
| L2-1 | 反俗套过滤：核心设定避免"AI统治""外星入侵"等老套桥段，或有实质性颠覆 | 俗套库对比 |
| L2-2 | 章节间情节转承逻辑自洽，无断裂 | 叙事链检查 |
| L2-3 | 角色动机与核心冲突紧密关联 | 角色-冲突矩阵检查 |
| L2-4 | 世界观设定内部一致（科技/社会/物理规则无矛盾） | 设定一致性检查 |
| L2-5 | 术语使用统一（同一概念不出现多种称呼） | 术语一致性扫描 |
| L2-6 | 商业吸引力要素明确（存在感情线/悬疑/社会议题等至少一项） | 元素清单检查 |

## L2 新增测试项（Iter-01 QA 后追加）

| ID | 验收项 | 验证方法 |
|----|--------|----------|
| L2-7 | 术语跨文档一致性：所有术语在三份交付物中定义一致且无遗漏 | 自动扫描三份交付物术语列表，标记仅出现在单一文档中的术语 |
| L2-8 | 科学锚点文献精确度：论文引用的年份、作者、关键数据（如基因数）与原文一致 | 逐条核对科学声明中的文献引用 |
| L2-9 | 角色年龄-目标读者匹配度：核心角色年龄分布与目标读者画像匹配 | 检查核心角色年龄分布，标记代入感风险 |
| L2-10 | 商业元素覆盖度：包含≥3种商业吸引力元素（悬疑/感情/社会议题/视觉奇观/系列化钩子） | 元素清单检查 |

## L3 卓越标准

| ID | 验收项 | 验证方法 |
|----|--------|----------|
| L3-1 | X-Factor 具备"病毒传播"潜力，能用一句话概括吸引读者 | 电梯演讲测试 |
| L3-2 | 角色具备多维度深度，超越单一原型 | 角色复杂度评估 |
| L3-3 | 大纲可独立阅读理解，无需额外说明 | 可读性测试 |
| L3-4 | 包含潜在续作/系列化空间 | 检查开放线索 |
